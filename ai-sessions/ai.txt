# Assistant

The owner should understand the property's state within a few seconds.

I'd recommend including:

Property image
Property name
Location
Number of bedrooms
Progress bar (e.g. 70%)
Current onboarding status
Number of outstanding action-required items
Target go-live date

Avoid showing every checklist item on the card—reserve those for the detail view.

Use colour sparingly, with action-required items standing out most.

Session 4 – Edge Cases
# User

what edge cases can you spot in the dataset

# Assistant

A few stand out immediately:

One property has no image.
Display a placeholder image or branded fallback.
One property has an empty steps array.
Avoid dividing by zero when calculating progress.
Some property names are extremely long.
Ensure cards wrap text or truncate cleanly.
Notes vary considerably in length.
Allow wrapping rather than fixed-height containers.
Status values may expand in the future.
Consider a default badge style for unknown statuses.
Some properties have multiple action-required items.
Surface this count clearly in the overview.
